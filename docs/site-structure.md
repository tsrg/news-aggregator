# Структура регионального новостного сайта

Документ описывает информационную архитектуру и обязательные элементы сайта. Регион фиксирован (задаётся в конфиге), выбор пользователем недоступен.

## Карта сайта (URL)

| URL | Назначение |
|-----|------------|
| `/` | Главная: блоки «Главное», «Новости региона», «Общие новости», разделы по темам |
| `/section/region` | Все новости региона |
| `/section/[slug]` | Раздел по теме (политика, спорт и т.д.) |
| `/news/[id]` | Страница новости |
| `/[slug]` | Статические страницы (about, contacts, privacy и т.д.) |

## Главная страница (блоки сверху вниз)

1. Шапка — логотип, меню (Главная, Новости региона, разделы), без выбора региона.
2. Хлебные крошки — Главная на главной; на остальных: Главная → Раздел/Регион → [Статья].
3. Блок «Главное» (Top Stories) — 1–4 материала (регион + общие), один крупный + карточки.
4. Блок «Новости региона» — лента с `region=<фиксированный_регион>`, ссылка «Все новости региона» на `/section/region`.
5. Блок «Общие новости» — 3–5 материалов без региона или раздел `general`.
6. Разделы по темам — табы/блоки с последними по каждому разделу, ссылки на `/section/[slug]`.

## Разделы (Section) — slug и названия

| slug | Название |
|------|----------|
| `top` / `main` | Главное |
| `region` | Новости региона |
| `politics` | Политика |
| `society` | Общество |
| `incidents` | Происшествия |
| `sport` | Спорт |
| `culture` | Культура |
| `economy` | Экономика |
| `general` | Общие новости |

## Статические страницы (Page)

| slug | Название | Назначение |
|------|----------|------------|
| `about` | О проекте / О нас | Описание издания, миссия, история |
| `contacts` | Контакты | Адрес, телефон, email |
| `advertising` | Реклама | Условия размещения рекламы |
| `privacy` | Политика конфиденциальности | Персональные данные, cookies |
| `terms` | Правила использования | Условия использования сайта |
| `editorial` | Редакция | Состав редакции, как предложить новость |
| `rss` | RSS | Ссылки на RSS-ленты |

Минимально обязательные: **about**, **contacts**, **privacy**.

## Регион

Один на весь сайт, задаётся в `runtimeConfig.public.region` (или env `NUXT_PUBLIC_REGION`). В запросах к API передаётся параметр `region`. Переключателя в интерфейсе нет.
